pub const PBKDF2_ITERATIONS: usize = 100_000;

Number of PBKDF2 iterations to perform. The more iterations, the more difficult to try to compute a rainbow table to try to reverse the hash. However, more iterations also take more CPU cycles to compute.

We default to use 100_000 iterations which gives a reasonably large number of iterations to hinder a possible attacker.

As computing power increases, the number of iterations should also be increased to ensure the difficulity (in computing time) for an potential adversay to try to compute a rainbow table for the PBKDF2 + HMAC-SHA-256 combination.
